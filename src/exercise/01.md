### 1) Attribute vs Properties

Attributes are the one we give in html elements. For example: id, class, custom
attributes (non standard html attributes)

Dom converts standard attributes into javascript object and attach it to element
object. For instance, if the tag is `<body id="page">`, then the DOM object has
`body.id="page"`.

If an attribute is non-standard, there won‚Äôt be a DOM-property for it. All
attributes are accessible by using the following methods:

- elem.hasAttribute(name) ‚Äì checks for existence.
- elem.getAttribute(name) ‚Äì gets the value.
- elem.setAttribute(name, value) ‚Äì sets the value.
- elem.removeAttribute(name) ‚Äì removes the attribute.

### 2) innerText vs textContent

`Prefer using textContent over innerText`

innerText - ignores styles and hidden content

textContent - displays hidden content and styles also.

## Background

It doesn't take long to learn how to make "Hello World" appear on the page with
HTML:

```html
<html>
  <body>
    <div>Hello World</div>
  </body>
</html>
```

The browser takes this HTML code and generates
[the DOM (the Document Object Model)](https://developer.mozilla.org/en-US/docs/Web/API/Document_Object_Model/Introduction)
out of it. The browser then exposes the DOM to JavaScript so you can interact
with it to add a layer of interactivity to your web-page.

```html
<html>
  <body>
    <div>Hello World</div>
    <script type="module">
      // your JavaScript here
    </script>
  </body>
</html>
```

Years ago, people were generating HTML on the server and then adding JavaScript
on top of that generated HTML for interactivity. However, as requirements for
that interactivity became more challenging, this approach produced applications
that were difficult to maintain and had performance issues.

So modern JavaScript frameworks were created to address some of the challenges
by programmatically creating the DOM rather than defining it in hand-written
HTML.
